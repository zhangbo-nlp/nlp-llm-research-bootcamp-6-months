# Week 01 - Python, Git and Research Habits

## 本周定位

这是科研训练的启动周。目标是让学生建立最基本的编程环境、版本管理习惯和科研记录习惯。零基础学生本周最重要的成果是“能独立运行、修改、保存、提交一段代码”。

建议投入：8-10 小时。

## 本周目标

- 理解科研学习和课程学习的区别：科研强调问题、证据、复现和表达。
- 能使用 Python 解释器或 Jupyter/Colab 运行代码。
- 能使用 Git 记录修改，理解 commit 的意义。
- 能写 Markdown 周报。
- 建立个人学习仓库和资料收藏方式。

## 理论学习

1. Python 是什么  
   重点理解解释执行、变量、类型、表达式、函数调用、错误信息。

2. 版本控制是什么  
   重点理解 repository、commit、working tree、staging area、remote。

3. 科研记录是什么  
   重点理解：做了什么、为什么做、如何复现、结果是什么、下一步是什么。

4. NLP/LLM 的整体地图  
   只需要知道关键词：文本、token、模型、训练、推理、评测、数据集、论文。

## 工具学习

- 安装或熟悉 Python 3.12+。
- 使用一种 notebook 环境：JupyterLab、VS Code Notebook 或 Google Colab。
- 安装并配置 Git。
- 注册 GitHub 或使用学校代码平台。
- 学习 Markdown 基本语法：标题、列表、代码块、链接、表格。

- Pro Git 第 1-2 章：https://git-scm.com/book/en/v2

## 实践任务

1. 创建个人学习仓库  
   仓库名建议：`nlp-llm-learning-2026`。创建以下目录：
   - `notes/`
   - `notebooks/`
   - `weekly_reports/`
   - `paper_notes/`

2. 完成第一个 Python notebook  
   文件名：`notebooks/week01_python_basics.ipynb`。内容包括：
   - 打印一句自我介绍。
   - 创建字符串、整数、浮点数、列表、字典。
   - 写一个函数 `count_words(text)`，统计英文句子的词数。
   - 故意制造一个错误，截图或复制错误信息，并解释错误原因。

3. 完成一次 Git 提交  
   至少包含：
   - `README.md`
   - `notebooks/week01_python_basics.ipynb`
   - `weekly_reports/week01.md`

4. 写第一份学习周报  
   使用 `templates/weekly_report_template.md`。必须回答：
   - 本周最陌生的概念是什么？
   - 哪个错误花费时间最多？
   - 下周想提前解决什么问题？

## 最低要求

- 能在本地或 Colab 运行 Python 代码。
- 能完成至少 2 次 Git commit。
- 能用 Markdown 写 100 字以上周报。

## 挑战任务

- 学会使用 `.gitignore` 忽略临时文件。
- 在 README 中加入学习路线表格。
- 用 GitHub Issues 记录 3 个本周遇到的问题。

## 验收标准

- 打开 notebook 后可以从头运行到尾。
- Git 历史中能看到清晰 commit message。
- 周报中有真实错误和解决过程，而不是泛泛总结。

## 自测问题

1. Python 中变量和字符串字面量有什么区别？
2. Git commit 记录的是“文件”还是“修改快照”？
3. 为什么科研代码必须记录依赖版本和运行方式？
4. notebook 的优点和缺点各是什么？
